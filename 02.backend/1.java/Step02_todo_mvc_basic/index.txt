Step01에서의 한계점
1. 하나의 클래스인 App.java에서 데이터의 입출력, 연산, 출력 처리를 모두 다 하고 있음.
-> Godclass
2. 객체가 아닌 기본 자료형을 그대로 활용 중이라 관리가 불편함.

Step02
1. Todo 데이터를 하나로 모아서 관리해주는 별도의 클래스 Todo 생성.(패키지는 com.todo.model)
- 하나의 필드 추가 : Boolean isCompleted // 진행여부

2. 생성한 클래스(Todo)를 활용하여 데이터를 관리한다.
3. 객체(Todo) 배열을 활용하여 읽은 데이터들을 Java에서 관리한다.
4. 할 일 순서 번호로 하나의 Todo를 조회하는 기능을 구현한다.

개선된 부분
1. 기본 자료형들을 하나의 참조 자료형(참조 타입, Todo)으로 모아서
관리하기 때문에 raw data(날 것 그대로의 데이터)를 건들지 않고, 별도의 클래스로 관리하게 됨(Wrapping).

현재 한계점
1. 메인 화면에 해당하는 App.java에서 데이터의 입출력과 결과 데이터 출력 등을 전부 수행하고 있음.
-> GodClass
